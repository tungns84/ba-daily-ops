<objective>
Add the new deterministic `ba-tools mermaid-render` command: extract the inline ```mermaid fence
from a diagram `.md`, write `diagram.mmd`, resolve the `mmdc` CLI via the locked 4-step chain,
invoke it via `subprocess` (list-form, no shell), and emit an image — hard-failing with
`BaToolsError` exit 2 (never a synthetic image) when no CLI resolves. Implements MMD-03 and
ROADMAP success criterion 3.

Purpose: This is the first render-capable spine route. It establishes the `mmdc` resolution +
hard-fail pattern that Phase 4 (`ba-mockup`) and deferred render plugins will mirror. The CLI
invocation is file/command/hash-provable work, so it belongs in `ba-tools`, not an agent shell-out
(determinism boundary, DESIGN §5/§11).

Output: `mermaid_render_cmd.py` (new module), an additive edit to `__main__.py`, a test module,
and two fixtures.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| caller → `mermaid-render` argv | Untrusted `--slug`, `--artifact`, `--mermaid-cli`, `--format` cross into path composition + subprocess |
| artifact `.md` content → `.mmd` | Untrusted fence body extracted from the diagram file and written to disk, then passed to mmdc by file path |
| `mermaid-render` → `mmdc` process | ba-tools spawns an external Node binary |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | Tampering | `--slug` path injection (e.g. `../../etc`) into output dir | mitigate | Compose `out_dir` then `is_within_root(out_dir, root)`; raise `PATH_TRAVERSAL` exit 2 on escape (render_cmd.py pattern) |
| T-03-02 | Tampering | `--artifact` path pointing outside repo root | mitigate | Resolve artifact path and confirm under root via `resolve_repo_root`/`is_within_root` before reading |
| T-03-03 | Elevation of Privilege | `--mermaid-cli` flag pointing to an arbitrary executable | accept | Trusted operator input — same trust level as the subprocess argv the operator already controls; documented, no remote/untrusted source supplies this flag |
| T-03-04 | Tampering | Fence body containing shell metacharacters reaching mmdc | mitigate | mmdc receives the `.mmd` file PATH via list-form argv, never the fence body as a shell string; `subprocess.run` list-form, never `shell=True` |
| T-03-05 | Denial of Service | concurrent writers clobbering `diagram.mmd` | mitigate | `FileLock(timeout=10)` guard on the `.mmd` write (render_cmd.py `_guarded_write`); `LOCK_TIMEOUT` exit 2 on contention |
| T-03-SC | Tampering | npm/pip/cargo installs | mitigate | No new package installed this plan (RESEARCH Package Legitimacy Audit: none); `@mermaid-js/mermaid-cli` invoked only via the fixed `npx -p @mermaid-js/mermaid-cli mmdc` form, never auto-installed |
</threat_model>
