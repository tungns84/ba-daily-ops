<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| operator arg -> scaffold | `init` operator string is untrusted; must key the static route table |
| `.ba-ops/config.json` -> flag reads | config file content is trusted input but must not be created/mutated implicitly |
| filesystem paths -> scaffold writes | scaffold writes must stay under repo root |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-1-01 | Tampering | `scaffold.ensure_scaffold` writes | mitigate | All scaffold writes are `root/".ba-ops"/...` derived from `resolve_repo_root`; no path arg from the user controls the target file names |
| T-1-04 | Tampering | `init` operator inference | mitigate | `init` validates the operator against the static `DEFAULT_ROUTES` table; unknown -> exit 2; no free-text route inference |
| T-1-10 | Tampering | implicit config default persisted | mitigate | `config.flag` uses `cfg.get(name, True)` and never writes a default to disk on absence (Anti-Pattern "Writing config.json on absence") |
</threat_model>
