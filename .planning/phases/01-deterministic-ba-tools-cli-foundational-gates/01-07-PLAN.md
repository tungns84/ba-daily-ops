<objective>
Lock the cross-cutting contracts that span every command and complete the byte-check
gate's second wiring layer. Two integration test files validate, across all ~16
subcommands: (1) the flat output envelope — success -> `{ok:true, failures:[], ...}` on
stdout, error -> `{ok:false, failures:[...]}` on stderr + exit 2, no nested `data` wrapper
(TOOL-13, CDX-05, D-03/D-04); and (2) path safety — every command resolves paths relative
to `--repo-root` and the package contains no hard-coded machine path, with subprocess
self-calls using `sys.executable` (TOOL-14, DESIGN §11). A committed git `pre-commit` hook
invokes `ba-tools byte-check` so the build fails locally before commit (GATE-04 layer 2,
D-05); the hook activates once the repo is under git but the subcommand remains the source
of truth (D-06).

Purpose: these contracts are only meaningfully verifiable once all command bodies exist
(Wave 1), so this Wave-2 plan spot-checks them holistically and finalizes the gate wiring.
Output: `test_output_contract.py`, `test_paths.py`, the `pre-commit` hook, and a short README.
</objective>
