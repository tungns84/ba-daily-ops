<objective>
Establish the Wave-0 foundation for the `ba-tools` deterministic CLI: the Python
package skeleton, the three shared infrastructure modules every command depends on
(`errors.py`, `output.py`, `repo.py`), the argparse dispatcher in `__main__.py` that
wires all 12 commands and the `BaToolsError` -> stderr -> exit-2 handler, the
`pyproject.toml` (pytest config + `filelock` runtime dependency + entry point), and
the complete pytest scaffold (`conftest.py` + 17 test files as stubs).

This plan installs the single runtime dependency (`filelock>=3.29.4`, per D-01/D-02)
and creates every command module as an importable stub so `python -m ba_tools --help`
succeeds before any command logic exists. Wave-1 plans then fill in the command bodies
against these fixed contracts (output envelope, error handling, repo-root path safety)
without a scavenger hunt.

Purpose: lock the output contract (TOOL-13, CDX-05) and path-safety contract (TOOL-14)
as shared code so all ~16 commands are consistent, and satisfy the VALIDATION.md Wave-0
gap list (filelock install + pyproject + conftest + 17 test files) so the Nyquist
sampling loop can run from the first Wave-1 task.
Output: an importable `ba_tools` package with stubbed commands, runnable test suite
(stubs skip/xfail), and `filelock` installed.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| CLI args -> filesystem | `--repo-root` and file path args are untrusted input that resolve to filesystem paths |
| stderr/stdout -> Codex chat | error JSON is read by a human/agent; must not leak stack traces or secrets |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-1-01 | Tampering | `repo.resolve_repo_root` / `is_within_root` | mitigate | `resolve_repo_root` returns an absolute resolved Path; `is_within_root` uses `Path.resolve().is_relative_to(root.resolve())` â€” Wave-1 file ops gate on it before any write |
| T-1-07 | Information Disclosure | `__main__.py` BaToolsError handler / `errors.py` | mitigate | Handler prints only `{ok:false, failures:[...]}` to stderr; `BaToolsError` carries a structured failures list with no traceback or credential content |
| T-1-SC | Tampering | `pip install filelock` (supply chain) | mitigate | `filelock` is `[ASSUMED]`-cleared in RESEARCH Package Legitimacy Audit (tox-dev org, 80+ versions, source repo `github.com/tox-dev/py-filelock`); pinned `>=3.29.4`; see Task 1 verify that imports the installed module |
</threat_model>
