<objective>
Implement the remaining deterministic utility commands: `extract-uc` (TOOL-10),
`template fill` (TOOL-11), `discovery add|list` (TOOL-12), `scan` (TOOL-15, advisory),
and `confirm` (GATE-02, pass-through in v1). `extract-uc` parses a `"<file>: ## UC-001.
<name>"` spec and returns the UC section + parsed identity using a level-aware heading
extractor (RESEARCH Pitfall 5 — stop only at a same-or-higher-level heading, never at a
deeper subsection). `template fill` does deterministic string substitution from
`ba-core/templates` and refuses an `--out` path that escapes the repo root (T-1-09).
`scan` runs an advisory prompt-injection pattern match that emits WARN findings and NEVER
blocks (exit 0 always — D-07/D-08, Open Decision #2). `confirm` is a v1 pass-through that
exits 0 (RESEARCH Open Question 2 — the confirm is an agent-level judgement in Codex chat).

Purpose: complete the deterministic CLI command surface so no later operator retrofits
these. All are file/command-provable; none make a judgement (scan flags, never decides).
Output: the 5 command modules + the shared section extractor + their tests.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| `--uc` / `--file` / `--out` paths -> filesystem | extract-uc source, scan target, template --out are untrusted paths |
| scanned file content -> findings | file content is untrusted (prompt-injection surface) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-1-09 | Tampering | `template fill --out` escape | mitigate | `--out` resolved under repo root and gated by `is_within_root`; escaping path -> PATH_ESCAPE exit 2 (RESEARCH Security Domain) |
| T-1-02 | Tampering | `scan` prompt-injection findings | mitigate | Advisory only (D-07): WARN findings, `blocked:false`, exit 0; output flagged for human review, never auto-fed back into agent prompts |
| T-1-01 | Tampering | extract-uc / scan source path read | mitigate | Source files resolved under repo root and gated by `is_within_root` before read |
</threat_model>
