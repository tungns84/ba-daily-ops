<objective>
Author the ba-uc conductor: the only operator that invokes others. It runs the
three spine operators as one sequential agent loop (srs-analyze full then mermaid
full then mockup full then index update), applies an operator-appropriate gate
after each step (full Quality gate after srs per D-G1; D-G2 index-integrity gate
after mermaid/mockup), writes pipeline status between steps (complete only after
the gate passes; failed on gate fail then STOP per D-RES1), threads one shared
slug captured from the srs step verbatim to every later step, and forwards
--fidelity to the mockup step. Routes: deliver (default) / resume / status /
iterate. Resume re-enters at uc-status next_step and re-runs that step from
scratch (D-RES2).

This is pure orchestration over the existing ba-tools surface and the existing
spine workflows — ZERO new ba-tools commands (RESEARCH Q4). The conductor mirrors
the Phase 4 ba-mockup skill/workflow/agent layout (PATTERNS). A static-check test
pins the workflow contract and the no-synthetic-render-on-spine invariant.

Purpose: Deliver UC-01 (one UC end-to-end), UC-02 (sequential loop + gate between
steps), UC-03 (resumable via uc-status; four routes, default deliver).
Output: ba-uc skill (2 files) + conductor workflow + conductor agent prompt + a
workflow-contract test.
</objective>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| user --uc / --fidelity input | Untrusted CLI args cross into the conductor and on to sub-operators |
| conductor → ba-tools subprocess | Conductor shells ba-tools (sys.executable, no shell=True); slug/path args cross here |
| conductor → .ba-ops/ writes | State, trace, and index writes cross the determinism boundary (lockfile-guarded) |
| conductor → sub-operator workflows | The only operator that invokes others; each step is a path reference, not embedded content |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-07 | Tampering | path-traversal via --uc / slug threaded to .ba-ops/ writes | mitigate | Slug is derived by the srs step (Phase 2 D-19) and all writes resolve via resolve_under_root / is_within_root (repo.py); the conductor passes the derived slug, never a raw user path — this IS the GATE-03 path-safety contract being honored on the spine |
| T-05-08 | Elevation of Privilege | subprocess injection on ba-tools invocation | mitigate | Conductor invokes ba-tools via sys.executable with argument vectors (no shell=True); workflow instructs argument-list invocation, never string-interpolated shell |
| T-05-09 | Tampering | concurrent conductor writes to STATE.md / INDEX.md | mitigate | All state and index writes go through acquire_state_lock (FileLock timeout=10); Plan 01 concurrent-write test proves no-clobber |
| T-05-10 | Spoofing | synthetic render slipping onto the spine | mitigate | test_no_render_cli_invoked_on_spine asserts no render-CLI step in ba-uc.md; GATE-03 contract (Plan 02) marks render plugin-only; spine drives authoring routes only |
| T-05-SC | Tampering | npm/pip/cargo installs | accept | No new dependencies; orchestration over existing ba-tools and stdlib pytest only — no install task |
</threat_model>
