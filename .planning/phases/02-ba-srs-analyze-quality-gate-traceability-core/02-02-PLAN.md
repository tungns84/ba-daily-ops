<objective>
Make `requirements.json` the gated canonical artifact and the rendered view its
deterministic output. Two deliverables:

1. Extend `ba-tools verify` to gate JSON directly (D-02/D-19/D-20): auto-detect
   `.json` or `--reqs-format json`, normalize the `source_trace` dict into the
   row shape the existing citation/lint pipeline already consumes, preserving
   every Phase-1 semantic (≥12-char verbatim, section-scoped, exit 2).
2. Add a deterministic JSON→Markdown renderer (`srs_render.py`) + a `render`
   command that emits the IEEE-830 `SRS.md` and the `REQUIREMENTS.md` registry
   FROM `requirements.json` — so the `.md` is a view, never the master (D-01).

Purpose: GATE-01 gates the canonical JSON; SRS-01/SRS-04 keep the JSON canonical
and the Markdown mechanically rendered (determinism boundary — rendering a known
schema is hash-provable, belongs in ba-tools).
Output: JSON-aware `verify`; `srs_render.py` + `render` command; evolved IEEE-830
`srs.md` template; verify fixtures F1-F4 + the verify test cases.

Review feedback incorporated (REVIEWS.md, this plan):
- **Codex HIGH — registry union (D-08).** `render_registry` now aggregates the
  UNION of all `.ba-ops/srs/*/requirements.json`; `render` is split into
  `render srs <slug>` (one SRS.md) and `render registry` (the global union). A
  single-slug render can no longer silently drop requirements owned by other
  slugs. Test `test_registry_union_all_slugs`.
- **Codex+OpenCode MEDIUM — source-doc plumbing.** verify resolves the cited doc
  from each requirement's `source_trace.doc`, NOT from CLI `--source`; the
  normalized row KEEPS the original `source_trace` dict so the dict-aware
  `check_grounding` (plan 02-01) reads it. Tests `test_source_trace_doc_used`
  and `test_source_trace_doc_mismatch` (a `--source` ≠ `source_trace.doc` case).
- **Codex+OpenCode MEDIUM — schema-invalid failure modes.** A `_validate_reqs_schema`
  pass rejects missing top-level `requirements`/non-list, a req missing `id`/
  `statement`, a `status` outside `{stated,derived}`, and a `stated` req missing
  `source_trace.span` — all via `BaToolsError` exit 2, never exit 1.
- **Codex MEDIUM — `section: null → document scope`.** Explicit
  `test_section_null_document_scope` covers the locked D-03 behavior.
</objective>
