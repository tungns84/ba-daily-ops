<objective>
Build the traceability spine's deterministic core — the two new `ba-tools`
commands that record artifact→REQ-ID provenance and rebuild the matrix:

1. `trace write` (TOOL-07, TRACE-04) — emit the uniform per-artifact trace
   record (D-05) into `.ba-ops/traces/<kind>-<slug>.json`, capturing
   `source_hash` (D-06) and per-REQ `statement_hash` (D-12), lockfile-guarded.
2. `index update` (TOOL-08, TRACE-05) — read ONLY the trace records (D-04),
   re-hash live source docs, and rebuild INDEX.md with gap/orphan/stale
   classification (D-08/D-13) plus the Status column and roll-up sections.

Purpose: This is the core-value spine — REQ-ID → source_trace → INDEX coupling.
It must be uniform-input (D-04) so Phases 3-5 (`ba-mermaid`/`ba-mockup`/`ba-uc`)
write the same record shape without `index update` ever learning their formats.
Output: `trace_cmd.py`, `index_cmd.py`, their registration, the F10
gap-orphan-stale fixture, the F12 non-convergence fixture skeleton, and tests.

Review feedback incorporated (REVIEWS.md, this plan — the highest-risk plan):
- **Codex HIGH — subset/orphan coverage.** `trace write` accepts caller-supplied
  `--req-ids FR-001,FR-002` (or `--req-ids-file`) so a downstream artifact can map
  a SUBSET of requirements and real orphans/gaps can exist. `--requirements` is
  used ONLY for statement-hash lookup; req_ids default to ALL requirements only
  when `kind=srs`. This unblocks the Phase 3/4 trace contracts.
- **Codex HIGH (security) — slug path-injection.** `kind`/`slug` validated against
  `^[a-z0-9][a-z0-9-]*$`; the output path is confirmed under root before write so
  a slug like `../../x` cannot escape `.ba-ops/traces/`.
- **Codex HIGH + OpenCode MEDIUM (security) — untrusted source_doc.** `index update`
  resolves each trace record's `source_doc` under repo root
  (`resolve_under_root`/`is_within_root`) before re-hashing — never
  `root / trace['source_doc']` directly.
- **OpenCode MEDIUM — shared hashing.** Both `trace_cmd.py` and `index_cmd.py`
  import `_sha256_file`/`_statement_hash` from `ba_tools/hashing.py` (created in
  plan 02-01) — no `index_cmd` → `trace_cmd` import, eliminating the circular risk.
- **Both MEDIUM — overwrite / missing-source / status precedence.** Overwriting an
  existing `<kind>-<slug>.json` fails `TRACE_EXISTS` exit 2 unless `--force`; a
  missing source_doc is reported `missing` in `## Stale` (not silently skipped);
  status precedence is `stale > gap > ok`.
</objective>
