<objective>
Build the traceability spine's deterministic core — the two new `ba-tools`
commands that record artifact→REQ-ID provenance and rebuild the matrix:

1. `trace write` (TOOL-07, TRACE-04) — emit the uniform per-artifact trace
   record (D-05) into `.ba-ops/traces/<kind>-<slug>.json`, capturing
   `source_hash` (D-06) and per-REQ `statement_hash` (D-12), lockfile-guarded.
2. `index update` (TOOL-08, TRACE-05) — read ONLY the trace records (D-04),
   re-hash live source docs, and rebuild INDEX.md with gap/orphan/stale
   classification (D-08/D-13) plus the Status column and roll-up sections.

Purpose: This is the core-value spine — REQ-ID → source_trace → INDEX coupling.
It must be uniform-input (D-04) so Phases 3-5 (`ba-mermaid`/`ba-mockup`/`ba-uc`)
write the same record shape without `index update` ever learning their formats.
Output: `trace_cmd.py`, `index_cmd.py`, their registration, the F10
gap-orphan-stale fixture, the F12 non-convergence fixture skeleton, and tests.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| `--artifact`/`--source-doc`/`--requirements` paths → trace write | attacker-influenced paths cross into reads/writes |
| trace record JSON → index update | trace records are parsed/trusted to build INDEX; malformed/oversized must not crash to exit 1 |
| concurrent operators → .ba-ops/traces/ and INDEX.md | parallel writes must not clobber shared state |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-07 | Tampering / Info Disclosure | path inputs to trace write | mitigate | resolve_under_root + is_within_root; PATH_TRAVERSAL exit 2; tested |
| T-02-08 | Tampering | malformed trace-record / requirements JSON | mitigate | json.loads in try/except → MALFORMED_JSON exit 2 (never 1) in both commands |
| T-02-09 | Tampering (no-clobber) | concurrent writes to traces/*.json + INDEX.md | mitigate | acquire_state_lock (FileLock timeout=10) on .json.lock / .md.lock siblings; LOCK_TIMEOUT on contention |
| T-02-10 | Tampering / Info Disclosure | LLM import smuggled into trace/index path | mitigate | grep assertion (G4); zero model-client import in ba_tools/ |
| T-02-11 | Spoofing | statement_hash / source_hash integrity | accept | SHA-256 collision computationally infeasible; stdlib hashlib is the correct primitive |
| T-02-SC | Tampering | npm/pip installs | mitigate | zero new dependencies this phase (D-18); nothing to install |
</threat_model>
